🐙 GitHub — Shared Database
PCO data is stored as a JSON file in a GitHub repo. All users read and write to the same file, so everyone stays in sync in real time.
1
Go to github.com/new and create a private repo (e.g. pco-tracker-data). Initialize it with a README.
2
Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic). Generate a token with the repo scope. Copy it.
3
Paste your details below and click Save & Test. The app will create a pco-data.json file in your repo automatically.
🌐 Host the App (Required for Team Sharing)
Browsers block API calls from local files. To share with your team and make GitHub sync work, you need to host this HTML file on GitHub Pages — it's free and takes 3 minutes.
1
Go to your pco-tracker-data repo on GitHub → click Add file → Upload files
2
Upload this pco-tracker.html file → rename it to index.html → commit it
3
Go to repo Settings → Pages (left sidebar) → under Source select Deploy from branch → main → / (root) → click Save
4
Wait 1–2 minutes → your app will be live at https://sameerpatel9.github.io/pco-tracker-data/ — share that URL with your team
⚠ Since this repo is Private, GitHub Pages won't work on the free plan. Either make the repo Public (the HTML file has no sensitive data — your token is only stored in your browser), or upgrade to GitHub Pro ($4/month).

Recommended: Make the repo public. The PCO data will be readable by anyone with the URL, but since it's internal operational data, this is usually acceptable. The GitHub token stays private in each user's browser.
⚡ Power Automate — Automatic Email Notifications
When a PCO stage deadline is missed, the app calls a Power Automate flow that sends an Outlook email to the owner automatically — no one needs to click anything.
1
Go to make.powerautomate.com → + Create → Instant cloud flow.
2
Choose "When an HTTP request is received" as the trigger. Set Method to POST. Use this JSON schema for the body:

{"type":"object","properties":{"toEmail":{"type":"string"},"ownerName":{"type":"string"},"pcoId":{"type":"string"},"pcoTitle":{"type":"string"},"stage":{"type":"string"},"deadline":{"type":"string"},"daysOverdue":{"type":"integer"}}}
3
Add an action: Send an email (V2) (Outlook). Set To = toEmail, Subject = [ACTION REQUIRED] PCO pcoId Overdue at stage, and fill the body with the dynamic fields.
4
Save the flow. Copy the HTTP POST URL shown in the trigger step and paste it below.
